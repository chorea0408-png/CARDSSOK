# 카드쑉(CARDSSOK) 서비스 소개서 / 인수인계서

> 최종 업데이트: 2026-07-14
> GitHub: `chorea0408-png/CARDSSOK` (main 브랜치 push → Vercel 자동 배포)

---

## 1. 한 줄 소개

**"원고만 붙여넣으면 30초 만에 카드뉴스가 완성되는, 로그인 없는 무료 웹 에디터."**

미리캔버스·망고보드·Canva 같은 범용 디자인 툴과 달리 템플릿을 고르는 과정 자체가 없다. 텍스트를 붙여넣는 순간 슬라이드가 자동으로 나뉘고 디자인까지 입혀진다.

---

## 2. 핵심 포지셔닝

| 항목 | 내용 |
|---|---|
| 핵심 차별점 | 로그인 불필요 + 템플릿 선택 없음 + 완전 무료. 경쟁사(미리캔버스/망고보드/Canva/Mirr/Tyle) 중 이 세 가지를 동시에 만족하는 곳이 없음 |
| 타겟 | 마케터, 1인 창업자, SNS 운영자 등 "빠르게 카드뉴스 초안 하나 뽑고 싶은" 라이트 유저 |
| 약점(의도적 트레이드오프) | 스톡 이미지 라이브러리 없음, AI 스타일 학습 없음, 영상/릴스 변환 없음 — 대신 가볍고 빠름 |
| 수익 모델 | Google AdSense (하단 배너 + 내보내기 팝업), 현재 완전 무료·프리미엄 티어 없음 |

## 3. 기술 스택

| 영역 | 선택 |
|---|---|
| 프레임워크 | React 19 + TypeScript + Vite 7 |
| 스타일 | Tailwind CSS v4 |
| 상태 관리 | Zustand v5 (`persist` 미들웨어로 localStorage 저장) |
| 이미지 캡처 | `html-to-image` (html2canvas는 사용 안 함, 제거 완료) |
| ZIP 생성 | `jszip` |
| 라우팅 | 별도 라이브러리 없이 자체 `pushState` 기반 미니 라우터(`src/router.tsx`) — `/`, `/app`, `/privacy` 3개 경로만 다룸 |
| 배포 | Vercel, `main` push 시 자동 배포. SPA 라우팅을 위해 `vercel.json`에 rewrite 설정 있음 |
| 테스트/CI | 없음 (자동화 테스트 0개, GitHub Actions 없음) — 알려진 리스크 |

## 4. 서비스 구조 (라우팅)

```
/           랜딩 페이지 (히어로 + 기능소개 + 3단계 사용법 + CTA + 푸터)
             → "무료로 시작하기" 클릭 시 /app 으로 이동
/app        실제 에디터 (원고 입력 → 슬라이드 생성 → 디자인 → 내보내기)
/privacy    개인정보처리방침 (AdSense 정책 대응, 로컬저장 방식 안내 포함)
```

에디터(`/app`)는 다시 두 상태로 나뉜다.

- **Empty State**: 처음 진입 시 3단계 온보딩 카드 + "예시로 먼저 볼까요?"(샘플 원고로 즉시 체험) 버튼
- **Editor**: 실제 작업 화면 — 데스크톱은 4분할(원고/슬라이드목록/캔버스/디자인) 좌우 패널, 모바일은 하단 탭 4개(원고/슬라이드/미리보기/디자인)로 전환

## 5. 핵심 기능

### 5-1. 원고 → 슬라이드 자동 변환
- 문단(엔터 두 번)마다 슬라이드 1장, 첫 줄=제목·나머지=본문
- 텍스트를 **붙여넣는 즉시**(확인창 없이) 자동 반영, "되돌리기" 토스트로 안전장치 제공
- 원고 키워드 기반 자동 테마 추천 (규칙 기반, AI 미사용) — "할인/이벤트" 계열은 피치톤, "일상/에세이"는 차분한 명조체, "데이터/분석"은 블루톤
- 슬라이드 생성 시 headline/body 길이에 따라 폰트 크기 자동 보정
- 첫 생성까지 걸린 시간을 실측해 "⚡ N.N초 만에 완성!" 토스트로 노출

### 5-2. 디자인 편집
- 테마 5종(단색/오로라/딥블루/피치선셋/다크글래스) × 폰트 4종, 색상·그림자·오버레이 세부 조정
- **"빠른 스타일 적용" 원클릭 프리셋 4종**(모던블루/다크시크/피치감성/미니멀화이트) — 전체 슬라이드 일괄 적용, undo 1회로 되돌리기 가능
- 3×3 텍스트 포지션, 배경/로고 이미지 업로드, 워터마크

### 5-3. 슬라이드 관리
- 드래그로 순서 변경(Pointer Events 기반), 복제, 삭제, 다중 선택 후 일괄 삭제
- Undo/Redo (표준 undo-stack/redo-stack 구조, 최대 30단계)
- 키보드 단축키: `⌘Z` 실행취소 / `⌘⇧Z` 다시실행 / `⌘D` 복제 / `N` 슬라이드 추가(모디파이어 없음 — `⌘N`은 브라우저 예약 단축키라 웹에서 막을 수 없어 대체) / `Delete` 삭제. Header의 키보드 아이콘 클릭 시 치트시트 모달로 확인 가능

### 5-4. 내보내기
- PNG(현재 슬라이드) / JPG(낱장) / ZIP(전체 일괄) / JSON(재편집·AI 연동용)
- 화질 1x/2x/3x 선택, 비율 1:1/4:5/9:16 선택
- 내보내기 클릭 시 광고 모달(300×250) 노출 후 진행 — 광고 수익 모델

## 6. 데이터 저장 (localStorage)

| 저장됨 (`cardssok-v1`) | 저장 안 됨 |
|---|---|
| appMode, project, slideRatio, exportQuality, selectedSlideId, slides(텍스트/디자인 설정) | 배경 이미지, 로고 이미지(용량 문제로 의도적 제외 — RightPanel에 경고 문구 있음) |
| 패널 너비(`cardssok-panel-widths`, 별도 키) | undo/redo 히스토리(세션 한정) |

로그인이 없으므로 **모든 데이터는 브라우저 1대에만 귀속**된다. 캐시 삭제·기기 변경 시 전부 유실됨(개인정보처리방침에도 명시).

## 7. SEO / 마케팅 자산

- OG 이미지(1200×630, 자체 제작) + Open Graph/Twitter 메타태그 — `index.html`에 `cardssok.vercel.app` placeholder 도메인으로 세팅되어 있음, **실제 도메인 확정 시 교체 필요**
- 개인정보처리방침 페이지 (`/privacy`) — AdSense 쿠키/맞춤광고 opt-out 안내, 문의처 포함
- robots.txt / sitemap.xml 없음 (미착수)

## 8. 지금까지 잡은 주요 버그 (참고용)

같은 원인 계열의 버그가 반복적으로 발견되어 함께 정리한다: **"슬라이드 배열이 통째로 바뀌는 동작(붙여넣기 재생성/undo/redo/삭제) 이후 `selectedSlideId`가 존재하지 않는 슬라이드를 가리켜 캔버스가 빈 화면이 되는 문제"** — 총 5곳에서 발견되어 모두 수정 완료 (다중선택 삭제, 단일삭제, undo/redo, 새로고침 복구, ZIP/JPG 내보내기 중 삭제).

그 외: undo/redo 인덱싱 자체가 한 칸씩 밀리던 근본 버그, 모바일에서 hover 전용이라 슬라이드 복제/삭제/드래그가 아예 불가능했던 문제, 좁은 데스크톱 창(768px 근처)에서 캔버스가 짜부라지던 문제 등을 수정했다.

## 10. 로컬 개발 / 배포

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # tsc + vite build → dist/
```

배포는 GitHub `main` 브랜치에 push하면 Vercel이 자동으로 처리한다. 별도 배포 명령이 필요 없다.

## 11. 파일 구조

```
src/
├── main.tsx              # 라우터 진입점 (/, /app, /privacy 분기) + ErrorBoundary
├── router.tsx             # 최소 pushState 라우터
├── App.tsx                # 에디터 레이아웃(데스크톱/모바일), Empty State, 키보드 단축키
├── store.ts                # Zustand 스토어 — 전체 상태/액션
├── types.ts                 # 타입 정의
├── pages/
│   ├── LandingPage.tsx      # "/" 랜딩
│   └── PrivacyPage.tsx      # "/privacy" 개인정보처리방침
└── components/
    ├── Header.tsx            # 상단 헤더 (비율/화질/내보내기/단축키)
    ├── ManuscriptPanel.tsx    # 원고 입력
    ├── LeftPanel.tsx          # 슬라이드 목록
    ├── CenterPanel.tsx        # 캔버스 미리보기
    ├── RightPanel.tsx         # 디자인 설정 + 빠른 스타일 프리셋
    ├── Toast.tsx              # 토스트 알림 시스템
    ├── ExportAdModal.tsx      # 내보내기 전 광고 모달
    ├── ShortcutsModal.tsx     # 단축키 치트시트
    └── ErrorBoundary.tsx      # 크래시 폴백 UI
```

`InputScreen.tsx`는 아무 데서도 import되지 않는 예전 프로토타입(죽은 코드)이며 정리 대상이다.
